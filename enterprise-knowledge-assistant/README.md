# Meridian — Enterprise Knowledge Assistant

Meridian is a complete, production-grade Enterprise Knowledge Assistant RAG (Retrieval-Augmented Generation) application designed for **AnthraSync Technologies Pvt. Ltd.** 

It leverages hybrid search (semantic dense embeddings + keyword lexical search) to retrieve information from corporate policy documents, FAQs, manuals, and compliance guides, then generates highly accurate answers with Groq's LLMs, complete with source citations and confidence metrics.

## Technology Choices & Rationale

- **FastAPI**: Extremely lightweight, fast asynchronous REST API framework. Native support for validation with Pydantic and autogenerated OpenAPI documentation.
- **Gradio**: Ideal for quick, interactive, and beautiful conversational web user interfaces. Offers built-in support for markdown rendering and customized styling interfaces.
- **ChromaDB**: Native Python vector database. Simple to configure, supports persistent disk storage, and offers fast cosine similarity metrics.
- **sentence-transformers (`all-MiniLM-L6-v2`)**: Standard, lightweight, yet powerful dense embedding model. Produces 384-dimensional vectors optimized for semantic search.
- **rank_bm25**: Python implementation of the Okapi BM25 ranking algorithm, providing excellent keyword matching capabilities to catch exact terms, IDs, or codes that semantic search might miss.
- **Groq API**:
  - `llama-3.3-70b-versatile` (Primary Model): State-of-the-art open-source LLM for factual extraction, reasoning, and context analysis. Highly cost-effective and low-latency via Groq's LPU.
  - `openai/gpt-oss-20b` (Rewriter Model): Fast, low-latency, and highly capable model for pronoun resolution and search optimization tasks.
- **reportlab**: Standard library for programmatically generating high-quality PDFs with headers, footers, tables, and exact page layouts.
- **pdfplumber**: A library optimized for layout extraction from PDFs. Preserves vertical structure much better than default PDF readers, enabling section header detection.

## Setup Instructions

### Prerequisites
- Python 3.11+
- A valid **Groq API Key** (Get one at [console.groq.com](https://console.groq.com/))

### 1. Installation
Clone or navigate to the project directory:
```powershell
cd enterprise-knowledge-assistant
```

Create a virtual environment and activate it:
```powershell
python -m venv .venv
.venv\Scripts\activate
```

Install the dependencies:
```powershell
pip install -r requirements.txt
```

### 2. Configure Environment Variables
Copy the `.env.example` file and fill in your Groq API key:
```powershell
copy .env.example .env
```
Open `.env` and update:
```env
GROQ_API_KEY=gsk_your_actual_key_here
```

## How to Run

### 1. Generate Sample PDFs
Run the document generator script. This script will programmatically create 6 multi-page corporate PDFs with custom headers/footers in the directory `data/documents/`:
```powershell
python scripts/generate_docs.py
```

### 2. Run Ingestion (Optional)
The ingestion process will read the generated PDFs, split them into sentence-boundary-aligned chunks of max 500 tokens (50 tokens overlap), generate embeddings, and populate ChromaDB:
```powershell
# You can run this programmatically by triggering the API endpoint,
# or run the evaluator/app which will auto-ingest if ChromaDB is empty.
```

### 3. Start the FastAPI Server
Launch the backend REST API:
```powershell
uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```
The interactive API documentation is available at [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs).

### 4. Start the Gradio Web UI
In a separate terminal (with the virtual environment activated), start the chat interface:
```powershell
python ui/gradio_app.py
```
Open your browser and navigate to [http://127.0.0.1:7860](http://127.0.0.1:7860) to interact with Meridian.

## API Documentation

### `POST /ask`
Submit a question with session tracking.
- **Request Body**:
  ```json
  {
    "question": "What is our work from home policy?",
    "session_id": "session-123"
  }
  ```
- **Response**:
  ```json
  {
    "answer": "AnthraSync Technologies allows permanent employees to work from home for up to 2 days per week, subject to alignment with deliverables and manager's approval. Core operating hours are 10:00 AM to 4:00 PM IST (HR_Policy.pdf, Section 3: Flexible Work & Remote Working).",
    "sources": [
      {
        "document": "HR_Policy.pdf",
        "page": 2,
        "section": "Section 3: Flexible Work & Remote Working",
        "text": "Our Work From Home (WFH) policy allows employees to work remotely up to 2 days per week..."
      }
    ],
    "confidence": 0.84,
    "confidence_label": "High",
    "rewritten_query": "work from home policy remote days"
  }
  ```

### `POST /ingest`
Scans the `data/documents/` folder, clears the old index, and re-embeds all files.
- **Response**:
  ```json
  {
    "status": "success",
    "chunks_indexed": 32
  }
  ```

### `GET /documents`
Lists all documents in the database and their indexed chunk counts.
- **Response**:
  ```json
  [
    {
      "document": "HR_Policy.pdf",
      "chunks": 8
    },
    {
      "document": "Customer_FAQ.pdf",
      "chunks": 6
    }
  ]
  ```

### `GET /health`
Verifies API status and connectivity to the vector database.

## Evaluation Suite

The evaluation suite tests the RAG pipeline using 20 ground-truth questions and 4 out-of-domain control questions to ensure high-fidelity responses.

To run the evaluation:
```powershell
python app/evaluator.py
```

### Metrics Calculated:
1. **Source Citation Accuracy**: Verifies that the correct document is cited in the response.
2. **Answer Keyword Relevance**: Computes the percentage overlap of expected keywords in the generated answer.
3. **Hallucination Rate**: Verifies that the model correctly fails-safe (returns "not found" with Low confidence) for out-of-domain questions instead of inventing information.

The evaluation outputs a detailed report at `tests/eval_report.json` and prints a summary table in the terminal.

## Known Limitations & Future Improvements

1. **In-Memory BM25 Corpus**: The current BM25 index is generated in memory by fetching all documents from ChromaDB on startup/query. While fine for thousands of chunks, a larger knowledge base (tens of thousands of documents) would require indexing BM25 into a persistent database (e.g., Elasticsearch or ChromaDB's native BM25 plugin if available).
2. **Simple Document Parsing**: Extremely complex tables or embedded charts in PDFs may require multi-modal parsing (OCR or layout-aware parsers like unstructured) rather than layout-text extraction.
3. **Stateless FastAPI Memory**: Session memory is stored in an in-memory dictionary. Production applications should store session history in Redis or a relational database for scalability.
